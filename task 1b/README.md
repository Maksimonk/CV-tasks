# Отчёт к заданию 1б

## 1. Влияние размера выборки на визуальное качество облака точек
**Чем больше точек** → тем подробнее и «плавнее» выглядит объект.  
**Чем меньше точек** → снижается детализация, могут теряться мелкие геометрические признаки.

## 2. Влияние размера вокселя в `voxel_grid_subsampling` на детализацию
- **Меньший размер вокселя** → выше детализация  
- **Больший размер вокселя** → сильнее агрегация точек, больше потери мелких деталей  
- При фиксированном количестве исходных точек: увеличение `voxel_size` уменьшает число выходных точек (больше точек попадает в один воксель)

## 3. Сравнение методов по времени выполнения

| Метод                     | Временная сложность | Примерное время |
|---------------------------|-------------------|----------------|
| `random_subsampling`      | O(N)              | 0.003 с        |
| `voxel_grid_subsampling`  | O(N) / O(N log N) | 0.161 с        |
| `farthest_point_sampling` | O(N·k)            | 113.872 с      |

**Вывод по производительности:**  
- `random_subsampling` — самый быстрый  
- `voxel_grid_subsampling` — умеренная скорость, приемлем для большинства задач  
- `farthest_point_sampling` (FPS) — значительно медленнее, особенно при больших N и k

## 4. Сравнение методов по сохранению геометрии объекта

### Random Subsampling
**Преимущества:**  
- Прост в реализации и очень быстрый  
- Сохраняет глобальную статистику распределения точек  

**Недостатки:**  
- Может пропустить редкие, но важные геометрические признаки (острые края, узкие кривые)  
- Не гарантирует равномерное покрытие поверхности  

### Voxel Grid Subsampling
**Преимущества:**  
- Обеспечивает равномерную локальную агрегацию  
- Снижает шум и переплотнённость  
- Детализация явно задаётся параметром `voxel_size`  
- Центроид-агрегация улучшает локальную геометрию  

**Недостатки:**  
- Удаляет детали, меньшие размера вокселя  
- Может искажать форму на тонких структурах (края, пластины)  

### Farthest Point Sampling (FPS)
**Преимущества:**  
- Максимизирует пространственное покрытие  
- Хорошо сохраняет контуры и крайние точки  
- Обеспечивает равномерное покрытие поверхности (даже при малом k)  

**Недостатки:**  
- Высокая вычислительная сложность  
- Не учитывает нормали или геометрические метрики поверхности (только расстояния в 3D)  

## Рекомендации по выбору метода

| Сценарий                                      | Рекомендуемый метод           |
|-----------------------------------------------|-------------------------------|
| Быстрое уменьшение выборки                    | `random_subsampling`          |
| Равномерное прореживание с контролем детализации | `voxel_grid_subsampling`  |
| Максимальное сохранение геометрии при сильном уменьшении | `farthest_point_sampling` |
| Обработка больших облаков точек               | `random_subsampling` или `voxel_grid_subsampling` |
| Сохранение контуров и формы                   | `farthest_point_sampling`     |
| Баланс скорости и качества                    | `voxel_grid_subsampling`      |
